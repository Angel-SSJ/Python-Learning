Puntos Claves
- Las pruebas unitarias son un método de prueba de software mediante el cual se someten unidades individuales de código
  fuente a diversas pruebas para determinar si son aptas para su uso
- Las pruebas unitarias hacen que tu código esté preparado para el futuro, ya que anticipas los casos en los que tu
  código podría fallar o producir un error
- Una unidad puede dividirse en varias categorías:
    Un módulo entero
    Una función individual
    Una interfaz completa como una clase o un método.
- El marco de pruebas unitarias de Python se inspiró en JUnit de Java y tiene características similares a los principales
  marcos de pruebas unitarias de otros lenguajes.
